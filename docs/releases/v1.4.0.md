# PickMeUp v1.4.0

发布日期：2026-07-11

## 亮点

- 移除图片 OCR，入口精简为粘贴短信 / Android 短信导入
- 删除与确认取件共用 3 天软状态防重，手动删除后不会立刻被短信重新导入
- 识别门控加强，抑制「运单在途 / 快到了」类误入库；支持驿小哥「可凭…到店提取」短信
- 列表增强：搜索、快递筛选、多选批量操作、点击复制取件码；删除卡片滑出动画
- 接入主流快递方形 SVG logo，卡片展示更清晰

## 变更内容

### 识别与数据

- 删除 `tesseract.js` 与 `ocrService`，包体积更小
- 包裹生命周期：`active` / `picked` / `deleted`，`expiresAt` 统一 3 天清理
- 取件码优先匹配「可凭 / 取件码」段式码，避免把「快递0366」尾号当取件码
- 新增驿小哥到店提取模板；负向拦截运单号、即将送达等在途提醒
- 快递 logo 映射：`services/courierIcons.ts` + `assets/icons/*.svg`

### 交互与界面

- 新增搜索栏、快递筛选、多选批量已取 / 批量删除
- 取件码一键复制；空状态按待取 / 历史 / 无结果区分
- 删除确认后卡片向右滑出，列表 layout 补位
- 引导页与 README 同步「仅短信 / 粘贴」能力描述

### 工程

- 版本号升级至 `1.4.0`
- 设计说明：`docs/superpowers/specs/2026-07-11-pickup-ux-and-recognition-design.md`
- 识别样例脚本：`scripts/run-extract-check.mts`

## 已知说明

- 短信读取与自动导入仍仅在 Android 原生环境可用
- 删除 / 已取件记录在本地保留 3 天防重，过期后可再次导入相同取件码
- GitHub Release 默认提供源码归档；如需附加 APK，请本地签名构建后上传

## 建议验证

1. 粘贴驿小哥顺丰 / 极兔「可凭 x-x-xxxx 到店提取」短信
2. 删除后再次粘贴同一短信 → 3 天内不入库
3. 粘贴「运单号 + 即将送达 / 快到了」→ 应拒绝
4. 搜索、多选、复制取件码、删除动效
